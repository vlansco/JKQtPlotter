87 /** \brief This class manages data columns (with entries of type \c double ), used by JKQTPlotter/JKQTBasePlotter to represent data for plots
88  * \ingroup jkqtpdatastorage
89  *
90  * \see \ref JKQTPlotterBasicJKQTPDatastore for a detailed description of how to use this class for data management!
91  *
92  *
93  * \section jkqtpdatastore_column_management Column Management
94  *
95  * \subsection jkqtpdatastore_column_management_generation Column Generation/Deletion
96  *
97  * A JKQTPDatastore manages a set of data columns. Each column is a continuous vector of numbers. Optionally it can be interpreted
98  * as a 2D image. In the latter case, the data is assumed to be ordered in row-major ordering (i.e. row for row).
99  *
100  * You can generate new columns e.g. by:
101  * - addColumn() which either generates internally managed (and extensible) columns, or accepts a simple \c double* pointer
102  * to external. In the latter case owner-ship may be transferred to the datastore, but can also retain externally.
103  * - addImageColumn() adds a column that should be interpreted as an image with given width and height (i.e. a row-major
104  * matrix with \c width columns and \c height rows.
105  * - copyColumn() duplicates an existing column
106  * - addCopiedColumn() copies an external dataset into the datastore. e.g. with code like:
107  * \code{.cpp}
108  * QVector<double> X, Y;
109  * const int Ndata=100;
110  * for (int i=0; i<Ndata; i++) {
111  * * const double x=double(i)/double(Ndata)*8.0*JKQTPSTATISTICS_PI;
112  * X<<x;
113  * Y<<sin(x);
114  * }
115  * plot.addGraph(linegraph=new JKQTPXYLineGraph(&plot));
116  * // by calling JKQTPDatastore::addCopiedColumn() the data is COPIED from the vector into the datastore
117  * linegraph->setXColumn(datastore->addCopiedColumn(X, "x"));
118  * linegraph->setYColumn(datastore->addCopiedColumn(Y, "y"));
119  * \endcode
120  * - addLinearColumn() adds a column with linearly increasing numbers (in a given range)
121  * - addLogColumn() and addDecadeLogColumn() add columns with logarithmically spaced values
122  * - addLinearGridColumns() adds two columns which represent x- and y- coordinates of points
123  * on a rectangular grid (useful for calculating image data)
124  * - addCalculatedColumn() calculates a column, based on row numbers and a C++ functor
125  * - addColumnCalculatedFromColumn() calculates a column, based on another column data
126  * - addCopiedMap() copies data from a std::map/QMap into two columns
127  * - ... several more functions for specific cases exist.
128  * - Also note that there are even library extensions that allow to import data directly from OpenCV matrices: JKQTPCopyCvMatToColumn()
129  * .
130  *
131  * Of course columns can also be deleted by calling:
132  * - deleteColumn()
133  * - deleteAllColumns()
134  * - deleteAllPrefixedColumns()
135  * - clear()
136  * .
137  *
138  * \subsection jkqtpdatastore_column_management_props Column Properties
139  *
140  * The properties of columns may be accessed using:
141  * - getRows() returns the number of rows in a specific column
142  * - getColumnPointer() returns a pointer to the data in the column
143  * - getColumnChecksum() calculated a checksum over the data in the column
144  * - getColumnNames() / getColumnName()
145  * .
146  *
147  * \subsection jkqtpdatastore_column_management_modify Modify Column Contents
148  *
149  * You can modify all data in a column by functions like:
150  * - setAll()
151  * - scaleColumnValues()
152  * .
153  *
154  * Also you can access a single entry from a column, using:
155  * - set() / get() set/read a column entry
156  * - inc() / dec() increment/decrement a column entry
157  * - appendToColumn() (adds a single row/new value to a column, if the column was not internally managed before, it will be copied into a new internal memory segment by the first call to this function!)
158  * - appendToColumns() (adds several values to several columns simultaneously, i.e. a shortcut to prevents writing several consecutive appendToColumn() for each column separately)
159  * - appendFromContainerToColumn() (adds several rows from a container to a column, if the column was not internally managed before, it will be copied into a new internal memory segment by the first call to this function!)
160  * .
161  *
162  * \subsection jkqtpdatastore_column_management_iterators Iterator Interface
163  *
164  * In addition to teh fucntions above, JKQTPDatastore also provides C++-style iteratos to read the data from a column:
165  * - begin()
166  * - end()
167  * .
168  *
169  * ... and also a \c std::back_inserter -style interface to append data to columns:
170  * - backInserter()
171  * .
172  *
173  * These allow to use the C++ standard algorithms to work with columns and also enabled the
174  * library \ref jkqtptools_math_statistics in this software package. You can use the functions
175  * above e.g. for code like:
176  *
177  * \code{.cpp}
178  * auto inserter it=datastore->backInserter(datastore->addColumn("new column"));
179  * for (auto it=datastore->begin(col1); it!=datastore->begin(col1) ++it) {
180  * *++inserter=sqrt(*it);
181  * }
182  * \endcode
183  *
184  * or simply
185  *
186  * \code{.cpp}
187  * // mean of a column in a JKQTPDatastore:
188  * double mean=jkqtpstatAverage(datastore1->begin(col1), datastore1->end(col1));
189  * \endcode
190  *
191  * Also there are functions to add data from iterator-defined ranges, e.g.:
192  * - addCopiedColumn()
193  * - addCopiedMap()
194  * - appendToColumn()
195  * .
196  *
197  * \see \ref JKQTPlotterBasicJKQTPDatastoreIterators
198  *
199  *
200  *
201  * \subsection jkqtpdatastore_column_management_images Image Column
202  *
203  * JKQTPDatastore stores the width and height of the represented image as metadata with any image column.
204  * This metadata is used to provide convenience access to the image pixels with:
205  * - setPixel()
206  * - getPixel()
207  * .
208  *
209  * The image width/height are read using:
210  * - getColumnImageWidth()
211  * - getColumnImageHeight()
212  * .
213  *
214  * This allows to write code like:
215  * \code{.cpp}
216  * for (int iy=0; iy<10; iy++) {
217  * for (int ix=0; ix<10; ix++) {
218  * datastore->setPixel(imgColumn, ix, iy, sin(ix*iy/30.0));
219  * }
220  * }
221  * \endcode
222  *
223  * \section jkqtpdatastore_dataio Data Output
224  *
225  * JKQTPDatastore provides several functions that allow to store its contents into files:
226  * - saveCSV()
227  * - saveSYLK()
228  * - saveMatlab()
229  * - saveDIF()
230  * .
231  *
232  * ... and function to read data into different data structures:
233  * - getData()
234  * .
235  *
236  *
237  *
238  *
239  * \section jkqtpdatastore_internals Internal Working
240  *
241  * This class manages a list if JKQTPDatastoreItem onjects that may each contain a chunk of memory, containig
242  * one or more columns of data. Each item can be accessed with get() by a specific ID which is returned by add().
243  * JKQTPColumn. You may only clear all chunks of memory/items. If you no longer need some of the data, but still want
244  * to access the rest you will simply have to destroy all JKQTPColumn that point to the item with their
245  * JKQTPColumns:datastoreItem property.
246  *
247  * \verbatim
248 
249 +- JKQTPDatastore ---------------------+ std::vector<JKQTPColumn>:
250 | | +- JKQTPColumn ----------------+
251 | +- JKQTPDatastoreItem --------+ | 0 | datastore |
252 | 0 | JKQTPSingleColumn |<--|-------------------|---datastoreItem = 0 |
253 | | 0 # # # # # # # # # # #<--|---|-------------------|---datastoreOffset = 0 |
254 | | | | | |
255 | +- JKQTPDatastoreItem --------+ | +- JKQTPColumn ----------------+
256 | 1 | JKQTPSingleColumn |<--|---\ 1 | datastore |
257 | | 0 # # # # # # # # # # #<--|---|-\ \--------------|---datastoreItem = 1 |
258 | | | | \----------------|---datastoreOffset = 0 |
259 | +- JKQTPDatastoreItem --------+ | | |
260 | : : | +------------------------------+
261 | : : | : :
262 | : : | : :
263 | +- JKQTPDatastoreItem --------+ | : :
264 |N-1 | JKQTPMatrixRow |<--|---\ : :
265 | | 0 # # # # # # # # # # # | | \ +- JKQTPColumn ----------------+
266 | | 1 # # # # # # # # # # # | | \ M-1 | datastore |
267 | | 2 # # # # # # # # # # #<--|---|--\ \------------|---datastoreItem = N-1 |
268 | | | | \---------------|---datastoreOffset = 2 |
269 | +-----------------------------+ | | |
270 | | +------------------------------+
271 +--------------------------------------+
272 
273 
274 \endverbatim
275  *
