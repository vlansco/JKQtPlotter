50 /*! \brief a speed optimized plotter class
51  \ingroup jkqtfastplotter
52 
53  This plotter class is (in contrast to JKQTPlotter) optimized for speed. It allows to plot function graphs/data in
54  a simple manner. The focus of design was not on a nice output (athough this is not unimportant), but on the speed
55  of the output. So the class may be used to di quick plotting of data, i.e. in fast image sequence applications.
56 
57  These measures were taken to increase speed:
58  - The coordinate system is drawn onto an internal image which is only updated, when the size of the widget
59  or a property of the coordinate system changes. When only the data changes, just the data plot has to be
60  updated.
61  - The class uses an internal buffer with the complete image, so a repaint does not always trigger a complete
62  repaint of all elements, but just measns painting the internal image to the screen. This also allows to
63  plot selections ... over the plot, without having to replot the plots.
64  - The definition of the coordinate system as well as the definition the system ticks ... is reduced to the absolute
65  minimum.
66  - The method setPlotUpdateEnabled() allows to prevent replotting of the class contents, e.g. when you want to set
67  a multitude of parameters without triggering a replot every time.
68  .
69 
70  The class supports these features:
71  - coordinate axes may be linear or logarithmic.
72  - ticks and tick labels are drawn by a simple algorithm: The user has to supply two values:
73  \c zeroTick is a tick value that exists in any case (i.e. 0) and \c tickDistance which contains the distance
74  between two ticks (only applicable to linear axes). Then the class starts to draw at zeroTick and move on to
75  \c zeroTick+tickDistance and \c zeroTick-tickDistance until \c min / \c max of the axis is reached. For logarithmic axes
76  the class uses \c zeroTick*10 and \c zeroTick/10
77  - plot may contain a grid (x and y grid may be switched on/off separately).
78  - plots are represented by descendents of JKQTFPPlot and the plotting code is completely
79  contained therein (and should be speed optimized).
80  - it is possible to keep a specified aspect ratio
81  - it is possible to couple different plots in a way, that they are plottetd with the same width/height
82  both in pixel and world coordinates by using plotterSizesChanged() and synchronizeX() / synchronizeY().
83  .
84 
85 */

397  /** \brief return x-pixel coordinate from x coordinate */
398  inline double x2p(double x) {
399  if (xAxisLog) {
400  if (x<=0) return xOffset+log(xMin/10.0)/JKQTPSTATISTICS_LN10*xScale;
401  return xOffset+log(x)/JKQTPSTATISTICS_LN10*xScale;
402  } else {
403  return xOffset+x*xScale;
404  }
405  }

407  /** \brief return x coordinate coordinate from x-pixel */
408  inline double p2x(long x) {
409  if (xAxisLog) {
410  return exp(JKQTPSTATISTICS_LN10*(static_cast<double>(x)-xOffset)/(xScale));
411  } else {
412  return (static_cast<double>(x)-xOffset)/(xScale);
413  }
414  }
415 
416  /** \brief return y-pixel coordinate from y coordinate */
417  inline double y2p(double y) {
418  if (yAxisLog) {
419  if (y<=0) return yOffset-log(yMin/10.0)/JKQTPSTATISTICS_LN10*yScale;
420  return yOffset-log(y)/JKQTPSTATISTICS_LN10*yScale;
421  } else {
422  return yOffset-y*yScale;
423  }
424  }
425 
426  /** \brief return y coordinate coordinate from y-pixel */
427  inline double p2y(long y) {
428  if (yAxisLog) {
429  return exp(JKQTPSTATISTICS_LN10*(static_cast<double>(y)-yOffset)/(-1.0*yScale));
430  } else {
431  return (static_cast<double>(y)-yOffset)/(-1.0*yScale);
432  }
433  }
