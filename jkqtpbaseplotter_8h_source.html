92 /** \brief base class for 2D plotter classes (used by the plotter widget JKQTPlotter)
93  * \ingroup jkqtpplotterclasses
94  *
95  * This class implements basic functionalities for the plotter classes. Those are:
96  * -# data storage using a JKQTPDatastore object
97  * -# coordinate transforms
98  * -# a set of properties for the graphs (colors, widthes ...) and also a system (see getNextStyle() to automatically
99  * choose a drawing style for different graphs.
100  * -# drawing the coordinate axes, grids ... (logarithmic and linear)
101  * -# saveing and printing the resulting plots
102  * .
103  *
104  * This class is NOT a widget, if you need a plotting widget, use JKQTPlotter. This class may be used to
105  * plot using a JKQTPEnhancedPainter.
106  *
107  * \see JKQTPlotter a QWidget class that displays a JKQTBasePlotter plot on a Qt Window.
108  *
109  *
110  * \tableofcontents
111  *
112  *
113  * \section jkqtplotter_base_datastore Data Storage
114  * As already mentioned this class does not provide means to draw graphs, but it contains a basic mechanism to associate
115  * data with it. This is imeplemented, using a JKQTPDatastore object together with some convenience access methods. No further
116  * data access is implemented and child classes are NOT forced to use this datastore. Hide it if you want. JKQTPlotter shows
117  * how to use it. This class implement a graph management, where graphs simply point to a set of columns inside the datastore
118  * that contain the actual plot data!
119  *
120  * If you call the \link JKQTBasePlotter::JKQTBasePlotter() constructor \endlink with no arguments, it will create an internal
121  * datastore object and you can start adding data by using getDatastore(). If you have an external JKQTPDatastore object you can
122  * give it as parameter to the constructor or use one of the other methods:
123  * - useExternalDatastore(): \copybrief JKQTBasePlotter::useExternalDatastore()
124  * - useAsInternalDatastore(): \copybrief JKQTBasePlotter::useAsInternalDatastore()
125  * - useInternalDatastore(): \copybrief JKQTBasePlotter::useInternalDatastore()
126  * - forceInternalDatastore(): \copybrief JKQTBasePlotter::forceInternalDatastore()
127  *.
128  *
129  *
130  *
131  * \section jkqtplotter_base_systems_baseplotter Coordinate Systems and Transformations
132  * These topics are discussed in the help for JKQTPCoordinateAxis. There is a set of coordinate transform
133  * methods (x2p(), y2p(), p2x(), p2y() which only call the respective methods in xAxis and yAxis objects.
134  *
135  *
136  * \section jkqtplotter_base_grids_baseplotter Axis Ticks and Grids
137  * - The plotting of coordinate axes and grids, as well as coordinate transforms is done by
138  * JKQTPCoordinateAxis descendents (see documentation there)
139  * .
140  * If you want to set the axis properties, use getXAxis() or getYAxis() to get a pointer to the axis objects which then
141  * may be used to set the axis properties.
142  *
143  *
144  * \section jkqtbaseplotter_appearance_and_style Appearance & Styling of the Graph
145  *
146  *
147  * \subsection jkqtplotter_base_plotsize Plot Sizes & Borders
148  *
149  * JKQTBasePlotter itself is no QWidget. It is merely tool to represent, manage and draw a graph onto any durface.
150  * JKQTPlotter then uses an internal JKQTBasePlotter instance to provide a QWidget.
151  *
152  * Still the JKQTBasePlotter needs a way to represent the location of the graph, the graph axes etc. inside
153  * a drawing rectangle (of size widgetWidth * widgetHeight ). The plot borders are then offsets to the plot
154  * indise that rectangele. This image illustrated the principle:
155  *
156  * \image html plot_widget_orientation.png
157  *
158  * When the plot is actually drawn, it may happen, that additional elements need to be positioned between the
159  * actual coordinate rectangle and the widget worder (e.g. axis labels, ticks, color bars etc.). Therefore
160  * this objects manages a set of internally calculated variables that also incorporate those dynamic elements.
161  * These are e.g. internalPlotBorderLeft, internalPlotBorderTop, internalPlotBorderRight, ... Their names all
162  * start with \c internal... .
163  *
164  *
165  * \subsection jkqtplotter_base_aspectratios Aspect Ratios
166  * First note that this functionality is only available and activated if both axes are linear!
167  *
168  * You can set two different aspect ratios:
169  * - The ratio of plotWidth/plotHeight (setAspectRatio(), setMaintainAspectRatio()) will keep the plots pixel-width and height at a certain value.
170  * \f[ \mbox{aspectRatio}=\frac{\mbox{plotWidth}}{\mbox{plotHeight}} \f]
171  * - The ratio of (xmax-xmin)/(ymax-ymin) (setAxisAspectRatio(), setMaintainAxisAspectRatio()) will keep the displayed axis ranges in a certain ratio.
172  * \f[ \mbox{axisAspectRatio}=\frac{\left|x_\text{max}-x_\text{min}\right|}{\left|y_\text{max}-y_\text{min}\right|} \f]
173  * .
174  * So to achieve different effects, use these combinations:
175  * - you have a 200x100 range where each 1x1-pixel should have an aspect ratio of 4:
176  * \code
177  * setAspectRatio(200.0/100.0);
178  * setAxisAspectRatio(4.0*getAspectRatio());
179  * \endcode
180  * .
181  *
182  *
183  * \subsection jkqtplotter_base_key Graph Keys (Legends)
184  * This class provides means to plot a key together with the functions. The plotting is only partially implemented in this base class, and has to be
185  * implemented by child classes, as no graph management, that could provide a title for a specific graph, is implemented here. Key Plotting is
186  * performed in the plotKey() method. This method basically draws a frame and background and then calls plotKeyContent() to draw the actual contents.
187  * The size of the key is determined by a call to getKeyExtent(), while it's position and plotting style is determined by the \c key_... parameters.
188  * If getKeyExtent() returns 0 for height or width, no key is plotted. The \c key_... parameters are designed to be rather common for any usual kind
189  * of key, so any implementation of key should be able to use them to parametrize the output. Note however that additional parameters might be defined
190  * in child classes.
191  *
192  * \image html plot_key.png
193  *
194  * As it may be convenient to position the key completely outside the graph this class may change the plotBorders to make the key fit outside
195  * SO if you use a keyPosition \b outside the graph you should NOT keep any space for the key in the margins, as the program will change them to
196  * accomodate the size of the key. If you select an \b inside key position the key will be plotted OVER the graph, i.e. the margins won't be changed
197  * Note that the margin change is internal and not visible in the class interface!
198  *
199  * There is also a possibility to determine the size of the key automatically, so all text fits in. This is activted by the property keyAutosize ( \copybrief JKQTPKeyStyle::autosize ). If this
200  * is \c true the function getKeyExtent() has to check the width of every key item and take it into account when calculating the width and height of the
201  * key content. By default this feature is switched ON.
202  *
203  *
204  * \section jkqtbaseplotter_dataexport_print Printing, Saving & Exporting
205  *
206  * \subsection jkqtplotter_base_saveprint Saving and Printing Graphs
207  * This class implement a set of function to save and print the graphs:
208  * - print() prints the graph on a QPrinter object
209  * - saveAsPixelImage() saves the plot into a pixel image file (PNG, TIFF, ... formats, as supported by Qt)
210  * - saveAsPDF() saves the graph as a PDF file (using the Qt printing engine)
211  * - saveAsSVG() saves the graph as a SVG file (using the Qt SVG library)
212  * - saveImage() saves the graph
213  * .
214  * You can also copy the contents of the plot into the clipboard:
215  * - copyPixelImage()
216  * .
217  *
218  * \subsection jkqtplotter_base_dataexport Exporting Graph Data
219  * This class implement a set of function to save the data of the graphs:
220  * - saveData() saves the data of the plot
221  * - saveAsCSV() saves the data of the plot as comma separated values
222  * - saveAsSYLK() saves the data of the plot as SYLK spreadsheet
223  * - saveAsDIF() saves the data of the plot as data interchange format file
224  * - saveAsMatlab() saves the data of the plot as a CSV file suitable for Matlab
225  * - saveAsSemicolonSV() saves the data of the plot as a Semicolon Separated Values (SSV)
226  * - saveAsTabSV() saves the data of the plot as a Tabulator Separated Values (CSV) file
227  * - saveAsGerExcelCSV() saves the data of the plot as a Text file (SSV) suitable for german excel, i.e. with comma as decimal separator
228  * .
229  * You can also copy the graphs' data into the clipboard:
230  * - copyData()
231  * - copyDataMatlab()
232  * .

235  * \section JKQTBASEPLOTTER_SYNCMULTIPLOT Synchronizing Several Plots
236  *
237  * Often a single plot is not sufficient, but several plots need to be aligned with respect to each other:
238  *
239  * \image html test_multiplot.png
240  *
241  * This can be achieved by putting several JKQTPlotter instances into a
242  * <a href="http://doc.qt.io/qt-5/layout.html">Qt Layout</a>. Then you can fill each plot differently and
243  * set the x-/y-range of each plot by hand. This method works for simple cases, but has several drawbacks:
244  * - Due to the independent and automatic layouting of each plot, the axes do not need to be aligned properly<br>
245  * \image html jkqtbaseplotter_synchronization_unequalaxes.png
246  * - When you print the plot, the printing does not know about the layout and the other plots in it. Therefor
247  * it will only print the plot itself.<br>
248  * \image html jkqtbaseplotter_synchronization_nogridprint.png "Printing with grid-printing-mode deactivated"<br>
249  * \image html jkqtbaseplotter_synchronization_withgridprint.png "Printing with grid-printing-mode activated"
250  * - when you zoom/pan in one of the plots (e.g. using the mouse), the other plots will not adapt their
251  * axes to match the new area, but especially in cases as in the image above it would be beneficial,
252  * that tha x-axis of the plot at the bottom follows the x-axis of the plot above etc.<br>
253  * \image html jkqtbaseplotter_synchronization_nonsyncedxrange.png
254  * .
255  *
256  *
257  * To overcome these limitations, JKQTPlotter (and JKQTBasePlotter) offer an API with which you can declare relations between
258  * different plots (one of them is made the master) and you can synchronize the axes of two plots, when
259  * zooming (also when calling e.g. zoomToFit() or setXY() ). This API is:
260  * - \ref JKQTBASEPLOTTER_SYNCMULTIPLOT_SYNC
261  * - \ref JKQTBASEPLOTTER_SYNCMULTIPLOT_GRIDPRINT
262  * .
263  *
264  * \subsection JKQTBASEPLOTTER_SYNCMULTIPLOT_SYNC Synchronizing Axis Range & Plot Width
265  *
266  * You can synchronize the plot width and axis range between two JKQTPlotter instaces using:
267  * - synchronizeToMaster() / JKQTBasePlotter::synchronizeToMaster() synchronizes the parent JKQTPlotter with another JKQTPlotter. With two boolean-parameters
268  * you can specify the axes to be synchronized. E.g. in the case above, you would call:
269  * \code
270  * // synchronize width/x-axis of plotResid to width/x-axis of plotMain
271  * plotResid->synchronizeToMaster(plotMain, JKQTBasePlotter::sdXAxis, true, true, true);
272  *
273  * // synchronize y-axis of width/plotResidHist to y-axis of width/plotResid
274  * plotResidHist->synchronizeToMaster(plotResid, JKQTBasePlotter::sdYAxis, true, true, true);
275  * \endcode
276  * This will synchronize the x-axes of the top (\c plotMain ) and bottom-left plot (\c plotResid ),
277  * as well as the y-axes of the bottom-left (\c plotResid ) and bottom-right plot (\c plotResidHist ).
278  * After this call they will have the same size in screen pixels and always span the same range
279  * in plot coordinates.
280  * - synchronizeXToMaster() / JKQTBasePlotter::synchronizeXToMaster() like synchronizeToMaster() / JKQTBasePlotter::synchronizeToMaster(),
281  * but synchronizes only the x-axes
282  * - synchronizeYToMaster() / JKQTBasePlotter::synchronizeYToMaster() like synchronizeToMaster() / JKQTBasePlotter::synchronizeToMaster(),
283  * but synchronizes only the y-axes
284  * - resetMasterSynchronization() / JKQTBasePlotter::resetMasterSynchronization() deletes all synchronizations
285  * from the JKQTPlotter
286  * .
287  *
288  * \see See \ref JKQTPlotterMultiPlotLayout for an extensive example of the functionality.
289  *
290  *
291  * \subsection JKQTBASEPLOTTER_SYNCMULTIPLOT_GRIDPRINT Grid Printing/Desclaring Relations between Plots
292  *
293  * Usually each JKQTPlotter/JKQTBasePlotter prints or exports only itself. But in many cases you might want to
294  * add several plots that form a grid layout and also want to export them as one image/print them on one page.
295  * To do this, JKQTPlotter/JKQTBasePlotter offers the <b>"grid printing" mode</b>. In this mode, you can declare
296  * relations between different JKQTPlotters/JKQTBasePlotters by putting them in a rectangular grid (like in a
297  * <a reh="http://doc.qt.io/qt-5/qgridlayout.html">QGridLayout</a>). then any export/print action will draw all (explicitly)
298  * decalred graphs.
299  *
300  * \image html jkqtplotter_gridprint.png
301  *
302  * The grid printing API is composed of these functions:
303  * - setGridPrinting() enables grid printing for this JKQTPlotter. If set to \c true , and you print afterwards,
304  * the printout (or export) will not only contain the plot itself, but also additional plots that were
305  * declared using addGridPrintingPlotter() (see below).
306  * - addGridPrintingPlotter() add a new plotter \a plotterOther for grid printing mode, at location \a x / \a y
307  * E.g. in the example shown above, you could call:
308  * \code
309  * plotMain->setGridPrinting(true);
310  * plotMain->addGridPrintingPlotter(0,1,plotResid);
311  * plotMain->addGridPrintingPlotter(1,1,plotResidHist);
312  * \endcode
313  * - setGridPrintingCurrentX() / setGridPrintingCurrentY() / setGridPrintingCurrentPos()
314  * sets the location of the calling plot inside the grid. <i>By default each plot assumes to be at (0,0).</i>
315  * - clearGridPrintingPlotters() clear all additional plotters for grid printing mode
316  * .
317  *
318  * \see See \ref JKQTPlotterMultiPlotLayout for an extensive example of the functionality.
319  *
